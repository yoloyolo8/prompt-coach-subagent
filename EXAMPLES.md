## 示例1: 实时优化模糊prompt

### 用户原始prompt:
```
研究Twitter推文内容
```

### Coach分析:
```markdown
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 Prompt质量分析
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

原始Prompt:
"研究Twitter推文内容"

质量评分: 42/100
├─ 清晰度: 8/25  (未指定研究目的和范围)
├─ 结构化: 10/25 (无组织结构)
├─ 上下文: 12/25 (缺少推文链接和背景)
└─ 效率性: 12/25 (需要至少2轮澄清)

❌ 主要问题:
1. 未明确研究什么方面(技术架构?传播分析?内容质量?)
2. 缺少具体的推文链接或内容
3. 无输出要求(报告?数据表?建议清单?)

💡 优化方案:

【方案1】结构化版本 (推荐)
"分析这条Twitter推文 https://twitter.com/xxx/status/xxx：
1. 项目基本信息(功能、技术栈、团队)
2. 传播数据(浏览量、点赞数、转发数、评论质量)
3. 病毒式传播的3个关键成功因素
4. 可复用的设计思路

输出: 800-1200字的Markdown分析报告,包含数据表格"

质量预估: 85/100 ⬆️ +43
全面覆盖分析维度,输出格式明确

【方案2】约束扫描版本
"用约束扫描框架深度分析推文项目:
<context>
推文链接: https://twitter.com/xxx/status/xxx
项目类型: AI创意工具/产品设计/其他
</context>

<task>
1. 建模: 构建产品传播的系统模型
   - 用户路径: 发现→了解→使用→分享
   - 价值链: 内容→功能→体验→社交货币
   
2. 主约束识别: 用户参与度的限制因素
   - 认知门槛(0=无门槛, 1=完全理解不了)
   - 使用成本(时间/金钱/学习曲线)
   - 分享动机(0=不想分享, 1=强烈想分享)
   
3. 参数化与极端化:
   - 认知门槛: →0时(全民可用) vs →1时(小众专业)
   - 使用成本: →0时(零成本体验) vs →∞时(高度定制化)
   - 分享动机: →0时(纯工具) vs →1时(社交属性强)
   
4. 决策区间: 找出病毒传播的临界点
   - 在什么参数组合下,产品从"普通"跃升为"病毒式传播"?
</task>

<o>
输出格式:
1. 约束分析表格(至少3个核心约束)
2. 极端化分析矩阵
3. 临界点识别(具体数值/状态)
4. 3个可行动的设计建议(基于约束分析)
</o>"

质量预估: 92/100 ⬆️ +50
深度匹配你的"约束扫描"思维框架,提供系统性洞察

【方案3】目标导向版本
"我想了解这个Twitter项目为何突然火爆,提取可复用的方法论:

目标: 找到3个可在我的产品中应用的创意要素
背景: 我正在做类似的AI工具产品设计
重点: 关注产品设计和传播机制,技术实现次要
推文: https://twitter.com/xxx/status/xxx

输出要求:
1. 核心创意的3个可迁移要素(每个100-150字)
2. 每个要素附上:
   - 在原项目中的具体体现
   - 为什么有效(心理学/用户行为分析)
   - 如何在我的产品中应用(具体建议)
3. 潜在风险提示(2-3点)

总长度: 1000-1500字"

质量预估: 78/100 ⬆️ +36
聚焦可行动的洞察,直接指向你的应用场景

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 用户选择:
```
用方案2,我想深度分析
```

### 结果:
- 一次性获得系统性分析
- 无需多轮澄清
- 输出高度结构化,便于后续使用

## 示例2: 批量分析模式

### 用户请求:
```
分析我最近20条对话,找出可以改进的地方
```

### Coach输出报告摘要:

```markdown
# Prompt质量分析报告
生成时间: 2024-12-16 10:30
分析范围: 最近20条对话 (2024-12-01 ~ 2024-12-16)

## 📈 总体统计
- 分析对话数: 20条
- 平均质量分: 67/100
- 质量分布:
  - 优秀 (80+): 3条 (15%)
  - 良好 (60-80): 9条 (45%)
  - 待优化 (<60): 8条 (40%)
- Token效率: 平均2.3轮达成目标

## 🎯 个人模式识别

### ✅ 你的成功模式

#### 模式1: 结构化+约束明确 (平均分: 92/100, 占比: 15%)
**特征**: 
- 使用5部分结构
- 每部分有明确要求
- 指定输出格式和长度

**典型案例**: 
- "60分钟快速入门:区块链量化交易" → 一次性完成,输出质量极高
  链接: https://claude.ai/chat/51caf19d-66b5-481c-9bea-227d6d573b3f

**为何成功**: 
清晰的框架让AI无需猜测,可以直接按结构填充内容。明确的约束(如"使用短句,无理论展开")确保输出风格统一。

#### 模式2: 约束扫描框架应用 (平均分: 88/100, 占比: 25%)
**特征**:
- 明确提出"用约束扫描分析"
- 包含建模→约束→参数化→极端化步骤
- 要求输出决策区间

**典型案例**:
- "思想实验的四步方法论" → 系统化产出,可复用性强
  链接: https://claude.ai/chat/594af4d3-f734-4488-b496-b445c02aea3f

**为何成功**:
这是你的核心思维框架,当显式应用时,AI能更好地对齐你的思考方式。

### ❌ 常见低效模式

#### 模式1: 目标模糊型 (平均分: 45/100, 占比: 40%)
**特征**: 
- "研究一下...", "帮我看看...", "分析这个..."
- 缺少具体分析维度
- 无输出要求

**典型案例**: 
- "研究Twitter推文内容" → 需要3轮澄清才明确需求
  链接: https://claude.ai/chat/f60ddbc9-dc50-4d7a-977e-5706871cce30

**问题诊断**: 
"研究"是开放式动词,可以指技术分析、内容评价、传播研究等多个方向。AI需要猜测你的真实意图。

**优化方向**: 
添加"研究什么方面"+"需要什么输出"。例如:"分析推文的传播机制,输出3个关键成功因素"

#### 模式2: 上下文缺失型 (平均分: 52/100, 占比: 30%)
**特征**:
- 直接贴链接/代码,无说明
- 假设AI知道你的目的
- 缺少背景信息

**典型案例**:
- [贴了一段代码] "这样对吗?" → AI不知道你的判断标准
  链接: [某个对话]

**优化方向**:
添加"为什么需要这个"+"期望达到什么效果"+"判断标准是什么"

### 🎨 个性化特征发现

**你的思维框架**:
- 约束扫描: 使用5次, 平均效果88/100 ⭐️ 这是你的强项
- 分层模块化: 使用3次, 平均效果82/100
- 60分钟快速入门: 使用1次, 效果92/100 ⭐️ 值得复用

**领域偏好**:
- 技术类prompt: 平均72/100 (12条)
- 系统分析类: 平均85/100 (5条) ⭐️ 你的优势区域
- 开放探索类: 平均48/100 (3条) ⚠️ 需要改进

**写作风格**:
- 结构化程度: 中-高 (当你刻意使用框架时达到"高")
- 简洁度: 适中 (不过于冗长,但有时缺少关键细节)
- 技术密度: 高 (使用大量专业术语,无需简化)

## 💡 改进建议

### 🚀 立即可用 (今天就能改善)
1. **在开放式探索前先定义"成功"**: 
   - 做法: 问自己"怎样算得到有用的答案?"然后写进prompt
   - 预期提升: +15-20分
   - 适用于: "研究..."、"分析..."类查询

2. **技术查询时主动提供3要素**: 
   - 做法: 目的 + 当前状态 + 期望结果
   - 预期提升: +10-15分
   - 适用于: 代码/系统/工具相关问题

### 📅 短期优化 (一周内培养习惯)
1. **复用"约束扫描"模板**: 
   - 当前使用率25%,但成功率100%
   - 可扩展到产品设计、技术选型、问题诊断等更多场景
   
2. **为常见任务建立快捷命令**:
   - 例如: `/analyze-tech [项目]` 自动套用技术分析模板

### 🎯 长期培养 (持续改进)
1. **建立个人"prompt规范"**:
   - 基于你的成功模式,形成check list
   - 复杂任务前快速自查
   
2. **追踪prompt迭代效果**:
   - 记录优化前后的对比
   - 每月运行Coach分析,看趋势变化
```

## 示例3: 持续学习与改进

### 第1周使用后
- 平均分: 67/100
- 主要问题: 目标模糊(40%)

- 技术查询时提供3要素
- 探索类问题用开放探索框架

### 第2周再次分析
- 平均分: 74/100 ⬆️ +7
- 目标模糊降至25%
- 新模板成功率: 85%

### Coach观察:
```markdown
## 改进追踪报告

恭喜!你的prompt质量在一周内提升了7分 🎉

**主要改进**:
- ✅ 约束扫描模板使用率: 25% → 40%
- ✅ 上下文完整性: 52 → 68
- ✅ 一次性达成率: 43% → 67%

**效果对比**:
| 场景 | 上周平均 | 本周平均 | 提升 |
|-----|---------|---------|------|

**最大突破**: 开放探索类(+16分)

**下一步建议**:
```

## 关键要点

通过这些真实示例可以看出:

1. **实时模式**立即见效
   - 从42分→85分(方案1)或92分(方案2)
   - 无需多轮澄清
   - 建议高度个性化(识别出约束扫描框架)

2. **批量模式**提供系统洞察
   - 识别个人强项(约束扫描)和弱项(开放探索)
   - 生成可复用模板
   - 追踪改进趋势

3. **持续学习**形成正向循环
   - Coach学习你的模式
   - 你应用Coach建议
   - 下次分析时看到进步
   - 进一步优化
