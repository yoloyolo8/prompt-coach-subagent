# Prompt模板库

## 模板1: 技术分析类

**适用场景**: 分析技术项目、代码、系统架构

**模板**:
```markdown
分析 [技术对象]:
<context>
- 目标: [分析目的]
- 背景: [为什么需要分析]
- 当前状态: [已知信息]
</context>

<task>
重点分析:
1. [分析维度1]
2. [分析维度2]
3. [分析维度3]
</task>

<constraints>
- 技术栈限制: [如果有]
- 性能要求: [如果有]
- 兼容性要求: [如果有]
</constraints>

<o>
- 格式: Markdown报告
- 长度: [字数/代码行数]
- 包含: [具体要素]
</o>
```

**实战示例**:
```markdown
分析这个React组件的性能问题:
<context>
- 目标: 找出导致页面卡顿的原因
- 背景: 用户反馈列表滚动时出现明显延迟
- 当前状态: 列表有2000+项,每项包含图片和多个交互按钮
</context>

<task>
重点分析:
1. 渲染性能瓶颈
2. 不必要的重渲染
3. 内存泄漏风险
</task>

<constraints>
- 技术栈限制: React 18, 不能引入新依赖
- 性能要求: 列表滚动FPS需达到60
- 兼容性要求: 支持Chrome, Safari, Firefox最新版本
</constraints>

<o>
- 格式: Markdown报告
- 长度: 800-1200字
- 包含: 问题诊断 + 3个优化方案(附代码示例) + 预期效果
</o>
```

## 模板2: 约束扫描分析类

**适用场景**: 复杂问题的系统性分析,需要找出关键约束和决策点

**模板**:
```markdown
用约束扫描框架分析 [问题/系统]:

<context>
问题描述: [具体问题]
领域: [所属领域]
</context>

<task>
按以下步骤分析:
1. **建模**: 构建[系统/问题]的抽象模型
2. **主约束识别**: 找出2-5个核心限制因素
3. **参数化**: 将约束量化为可调参数(定义范围)
4. **极端化分析**: 
   - 约束→0时会发生什么?
   - 约束→∞时会发生什么?
5. **决策区间**: 识别关键的cliff edges/leverage points
</task>

<o>
以表格形式输出:
| 约束 | 当前值 | →0的影响 | →∞的影响 | 临界点 | 可行动建议 |
|----|------|---------|---------|-------|----------|
| ... | ...  | ...     | ...     | ...   | ...      |

最后总结3个最重要的决策区间
</o>
```

**实战示例**:
```markdown
用约束扫描框架分析远程团队协作效率:

<context>
问题描述: 15人团队分布在3个时区,协作效率低于预期
领域: 团队管理
</context>

<task>
按以下步骤分析:
1. **建模**: 构建远程协作的信息流动模型
2. **主约束识别**: 找出核心限制因素
   - 时区差异(小时数)
   - 同步会议频率(次/周)
   - 异步沟通延迟(小时)
   - 文档完整度(%)
3. **参数化**: 
   - 时区差: 0-12小时
   - 同步会议: 0-10次/周
   - 响应延迟: 0-24小时
   - 文档质量: 0-100%
4. **极端化分析**: 
   - 完全同步(所有人同时区) vs 完全异步(12小时差)
   - 每天开会 vs 从不开会
5. **决策区间**: 识别效率跃变的临界点
</task>

<o>
以表格形式输出约束分析,并回答:
- 哪个约束改善能带来最大收益?
- 当前离最优区间有多远?
- 3个立即可行的优化建议
</o>
```

## 模板3: 快速学习类(60分钟入门)

**适用场景**: 需要快速掌握新概念、新技术、新领域

**模板**:
```markdown
60分钟快速入门: [主题]

请按以下5个部分组织:

1. **一句话定义** (含1-2个具体例子)
   - 用最简单的语言解释[主题]是什么
   - 给出2个真实世界的应用场景

2. **20个高频术语** (每个1-2句解释)
   - 列出最常见的专业术语
   - 简短解释含义
   - 标注常见误解

3. **工作流程图**
   - 用文字描述典型工作流
   - 标注每步的输入/输出
   - 指出关键决策点

4. **5个典型任务** (从简单到复杂)
   - 任务1: 入门级
   - 任务2-3: 进阶级
   - 任务4-5: 实战级
   - 每个任务附上完成步骤

5. **10个新手常见坑**
   - 列出最容易犯的错误
   - 每个附上避免方法
   - 标注严重程度(⚠️低 ⚠️⚠️中 ⚠️⚠️⚠️高)

要求:
- 使用短句,无理论展开
- 重实用性,轻学术性
- 总长度3000-5000字
```

**实战示例**:
见已有对话: "60分钟快速入门区块链量化交易"

## 模板4: 文档/代码修复类

**适用场景**: 修复、优化、重构现有内容

**模板**:
```markdown
[修复/优化/重构] [对象]:

<current_state>
[附上现有内容或描述当前状态]
</current_state>

<issues>
当前问题:
1. [问题1]
2. [问题2]
3. [问题3]
</issues>

<goals>
期望达到:
- [目标1]
- [目标2]
- [约束条件]
</goals>

<o>
输出:
1. 问题诊断总结
2. 修复后的完整版本(带注释说明改动)
3. 关键改进点说明(3-5点)
</o>
```

## 模板5: 探索研究类

**适用场景**: 开放式探索,需要多角度分析

**模板**:
```markdown
探索研究: [主题]

<context>
- 研究目的: [为什么要研究]
- 已知信息: [当前了解的内容]
- 重点关注: [最感兴趣的方面]
</context>

<approach>
研究路径:
1. [角度1]: [具体研究什么]
2. [角度2]: [具体研究什么]
3. [角度3]: [具体研究什么]
</approach>

<o>
每个角度输出:
- 核心发现(3-5点)
- 相关案例(2-3个)
- 可行动洞察(至少1个)

最后综合总结: 3个最重要的takeaways
</o>
```

## 使用建议

1. **选择模板**: 根据任务类型匹配模板
2. **填充内容**: 将具体信息填入占位符
3. **调整细节**: 根据实际需求调整约束和输出要求
4. **验证完整性**: 检查是否包含必要的上下文和约束
5. **预估效果**: 使用模板的prompt通常能达到80+/100分

## 自定义模板

如果你发现了新的高效模式,可以按此格式创建自己的模板:

```markdown
## 模板X: [名称]

**适用场景**: [描述]

**模板**:
```
[占位符版本]
```

**实战示例**:
```
[真实案例]
```

**预期效果**: [质量分/轮次]
```

然后保存到此文件或 `~/.claude/templates/` 目录
